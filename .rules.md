# Dashboard — Coding Rules

## Tech stack

- **Framework**: Next.js (App Router) + React
- **Data layer**: Refine + TanStack Table + React Query
- **ORM**: Drizzle ORM
- **UI**: shadcn/ui (`@/components/ui`)
- **Styles**: Tailwind CSS

## Icons (enforced)

**Only `@remixicon/react`** — no `lucide-react`, `react-icons`, or any other icon library.

```tsx
// ✅ correct
import { RiAddLine, RiBoxLine } from "@remixicon/react"

// ❌ forbidden
import { Plus } from "lucide-react"
```

## Component rules

- **Never modify `components/ui/*`** — treat as stable. Customize via `app/globals.css` or wrapper components.
- Use `@/components/ui` shadcn components, not raw Radix primitives.

### Page headers — use the right variant

| Page type          | Header             |
| ------------------ | ------------------ |
| List               | `ListViewHeader`   |
| Detail (read-only) | `ShowViewHeader`   |
| Create form        | `CreateViewHeader` |
| Edit form          | `EditViewHeader`   |

**Never put a `ShowViewHeader` on a create/edit page** — it injects stale detail actions.

Only pass `showEditAction` to `ShowViewHeader` when the page is read-only AND has a separate edit route.

### Buttons in headers

All buttons in the same toolbar must use the same `size`. For list page headers: `size="sm"`.

### Operation pages

- Use `DryRunActionButton` for all write operations.
- Destructive (irreversible): `variant="destructive"`. Reversible: `variant="default"`.
- Show `SqlViewer` in the plan preview when SQL is available.

### State visualization

Always use the state getter + tone map + `EntityStateBadge` pattern:

```tsx
const state = getItemState(record)
<EntityStateBadge label={state} tone={itemStateTone[state]} />
```

Never hardcode badge colors inline.

### Banned/destructive row highlighting

Use `getRowClassName` to add `bg-destructive/10` for banned/deleted rows:

```tsx
<DataTable table={table} getRowClassName={(r) => (r.deletedAt ? "bg-destructive/10" : "")} />
```

## Data layer rules

- **Resource name**: camelCase in code, kebab-case in URL path.
- **MySQL/Postgres note**: no `.returning()` in UPDATE — use `update().set().where()` + separate `select()`.
- Date-like fields (`createdAt`, `updatedAt`, `deletedAt`, etc.) are auto-converted to `Date` objects by `normalizeWriteValue`.

## Directory layout

```
app/dashboard/resources/   # list + show pages
app/dashboard/operations/  # write operation pages
components/ui/             # shadcn primitives (read-only)
components/refine-ui/      # Refine wrappers
components/operations/     # DryRunActionButton, SqlViewer
lib/refine/                # data providers
lib/{module}-records.ts    # state machines + formatters
db/{module}/               # schema + client per module
```
