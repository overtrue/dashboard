# Admin 目录重构实现计划

## 目标
将数据源、项目、用户、页面等管理功能统一组织到 `admin` 目录下，提供更清晰的系统架构。

## 已完成的工作

### ✅ Stage 1: 目录结构重组
**目标**: 创建新的 admin 目录结构
**成功标准**: 所有管理功能页面都在 admin 目录下
**状态**: 完成

**具体变更**:
- 创建 `apps/web/app/admin/` 目录
- 移动 `data-sources/page.tsx` → `admin/data-sources/page.tsx`
- 移动 `projects/page.tsx` → `admin/projects/page.tsx`
- 创建 `admin/pages/page.tsx` (页面管理功能)
- 创建 `admin/page.tsx` (管理概览页面)

### ✅ Stage 2: 菜单配置更新
**目标**: 更新侧边栏导航，指向新的 admin 路径
**成功标准**: 侧边栏中的管理菜单链接正确指向 admin 目录
**状态**: 完成

**具体变更**:
- 更新 `dynamic-menu-service.ts` 中的 URL 路径
- 添加 "管理概览" 菜单项，链接到 `/admin`
- 更新数据源、项目、用户、页面管理的路径

### ✅ Stage 3: 清理旧目录
**目标**: 删除旧的独立目录
**成功标准**: 旧的 data-sources 和 projects 目录被移除
**状态**: 完成

**具体变更**:
- 删除 `apps/web/app/data-sources/` 目录
- 删除 `apps/web/app/projects/` 目录

## 新的目录结构

```
apps/web/app/
├── admin/                    # 管理功能统一入口
│   ├── page.tsx             # 管理概览页面
│   ├── data-sources/        # 数据源管理
│   │   └── page.tsx
│   ├── projects/            # 项目管理
│   │   └── page.tsx
│   ├── users/               # 用户管理
│   │   └── page.tsx
│   └── pages/               # 页面管理
│       └── page.tsx
├── api/                     # API 接口
├── dynamic/                 # 动态页面
└── layout.tsx              # 根布局
```

## 功能特性

### 管理概览页面 (`/admin`)
- 提供到各个管理功能的导航卡片
- 清晰的视觉层次和图标设计
- 响应式布局，支持移动端

### 页面管理功能 (`/admin/pages`)
- 动态页面列表展示
- 发布状态管理
- 编辑和删除操作
- 加载状态和空状态处理

## 技术实现

### 组件复用
- 使用现有的 UI 组件库
- 保持与现有页面一致的布局和样式
- 面包屑导航支持层级结构

### 状态管理
- 使用 React hooks 管理本地状态
- 模拟数据展示功能
- 预留了真实 API 集成的接口

### 路由结构
- 清晰的 URL 层次结构
- 支持面包屑导航
- 便于权限控制和 SEO
